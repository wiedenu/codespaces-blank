IRH Franchise Application — Workflow Documentation

How the IRH online application flows through HubSpot: spam gating first, then concept-based rep routing. Two chained workflows, documented so anyone can see what runs, in what order, and why.

The two workflows, and how they connect

Every IRH online application hits the intake / spam gate first. Clean submissions are handed off to the routing workflow, which assigns the right rep by franchise concept. Spam is quarantined and never reaches routing.

IRH — Online Application: Intake & Spam Gate

Purpose

The front door for every IRH online application. It filters known spam out before routing, marks each contact's spam status, notifies the team of a real submission, and hands clean records to the routing workflow. Nothing reaches a rep until it clears this gate.

Logic — branches evaluated in order

The branch checks conditions top-down and stops at the first match. Spam is checked first; everything that doesn't match falls through to PASS.

  • ■End

Exclusions & edge rules

  • Spam filtering is manual / known-bad signals. Right now it catches the sender name "Derrick Young" and any phone containing "269" (269 = SW-Michigan area code, a known spam source). New spam patterns must be added by hand — this is not an adaptive filter.
  • Spam Status is the gate. Quarantine vs. Clean is what everything downstream keys off. Only Clean records continue to routing.
  • Step numbering is HubSpot's, not sequence. HubSpot numbers actions globally across branches (2, 3, 4, 5, 6), so "step 4" and "step 6" are the two branch ends running in parallel — not a single ordered list.
