High-value activity triggers to sync

A shortlist of marketing activities worth surfacing to reps as Salesforce Tasks. Not everything syncs — just the signals that say a human should look.

From John Wiedenheft To Cody Jandik Re Activity Sync build (parked since 6/26) Date July 7, 2026

The guiding rule: only fire a Task when the signal is strong enough that a rep should stop and look. Sync page views and email opens and reps mute the whole thing, and the real signals die with the noise. Everything below is scoped to what GreatAmerica can actually track today — a couple of items still need a feasibility check from you, flagged in amber.

Mechanism Native activity Workflow logic Status Ready Confirm w/ you
TIER 1

Hot — hand-raisers

Explicit intent. These should always sync.

“Contact Sales / Request a Quote” form submitted

Native activityReady

A direct ask for a human. The cleanest signal we have — maps straight to a form-submission activity.

Demo or meeting booked

Native activityConfirm w/ you

Self-scheduled time via a HubSpot Meetings link. Do reps actually book through HubSpot meeting links, or does scheduling live in Salesforce/Outlook? If the latter, this one drops.

Payment / financing calculator used

Workflow logicConfirm w/ you

Modeling a payment is deep intent. Two checks: is the calculator a marketing-owned web asset we can track, and does it sit behind the same wall as the application (which marketing can’t see)? If it’s walled off like the app, it drops too.

TIER 2

Warming

Pattern-based intent. Strong, but each needs a defined page set or workflow behind it.

High-intent page-view threshold

Workflow logicConfirm w/ you

Raw view counts are noise — three blog reads isn’t intent. The count only means something tied to which pages. Proposed rule:

IF 3+ views of high-intent pages within a rolling 7 days  OR  a return to one after 30+ days dormant → create Task High-intent pages = product / solution pages, the calculator, and contact / get-started pages. Built as a behavioral list or a “visited URL” workflow branch, not a native activity. Needs the page set defined jointly.

Bottom-funnel content consumed

Workflow logicConfirm w/ you

Case studies and vertical solution pages — the “this is a buyer, not a browser” set. Only works if we can name a specific list of those pages; we have no pricing guide to anchor it, so the set needs building.

High-intent email CTA clicked

Workflow logicReady

A click on the “talk to us / get started” link — a click, never an open. Filtered to specific CTA links so a generic click doesn’t trip it.

Dormant / closed-lost contact re-engages

Workflow logicConfirm w/ you

A previously cold or closed-lost contact does anything meaningful again. Reps value reopening these — the trigger is the state change, not the specific action.

EXCLUDE

Deliberately not synced

The noise that would train reps to ignore the sync.

  • Email opens, sends, deliveries
  • Single-page / generic page views
  • Newsletter or blog subscribe
  • Gated ebook / top-funnel downloads
  • List membership changes
  • Generic link clicks, social follows
  • Unsubscribes (route to ops, not a rep Task)

What I need from you

Before I build a test plan around this, a few things only you can answer on the Salesforce and build side:

  1. Build state. Where does Workflow 1 actually stand today? Still built-and-waiting from May, or has anything changed on the SF side since?
  2. Task mapping. Do these land cleanly as Salesforce Tasks, or is a custom Activity type / field cleaner? Any subject-line or due-date conventions reps expect?
  3. Record matching. How do we match a HubSpot contact to the right SF Lead/Contact and Account — existing sync ID, or something we need to define?
  4. Feasibility. Of the amber “Confirm” items, which are realistic to trigger from HubSpot on your side, and which are more trouble than they’re worth?
  5. Dedup. Can we enforce one open Task per contact per rolling window on the SF side, or does HubSpot need to gate it? (See below.)

One noise-control rule regardless of the list: cap it at one open Task per contact per rolling 3–5 days, surfacing the highest-value trigger rather than one Task per action. A hot contact doing five things in an afternoon should be one Task, not five. This single rule is the biggest driver of whether reps trust the sync.

Draft for feedback — nothing is built off this list yet. Honest read: once the amber items shake out, launch scope may be as tight as two Tier-1 triggers plus the page-view pattern. That’s a real finding about our trackable activity inventory, not a problem to paper over.
