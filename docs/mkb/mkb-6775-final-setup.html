What's Left to Set Up

  1. Create the "Review – Internal or External" field
    Single-select: Internal / External. Add to the relevant screens. Leave it optional — this is filled in by John only during the weekly pass, never by the team, so there's no required-field risk to the Marketing Intake Flow.
  2. Check for conflicting automations
    Quick look in Jira admin: confirm no existing rule already reacts to Flagged changing, so this doesn't become a second writer fighting another rule.
  3. Build the automation rule
    Trigger: due date field changes. Condition: Due Date Change Count > 2 AND status = Review. Actions: set Flagged = Impediment, then notify John. Test it on a throwaway ticket before turning it on for real ones — given what happened in March, don't skip this.
  4. Save the two JQL filters as dashboard gadgets
    Breach filter (External-only, SLA check): status = Review AND "Review - Internal or External" = External AND status CHANGED TO Review BEFORE -5d AND resolution = Unresolved Weekly worklist (for John's manual pass): status = Review ORDER BY updated DESC
  5. Put the weekly pass on the calendar
    Tie it to standup prep so it happens consistently — run the worklist filter, classify Internal/External, set the field on each.

    Write the final design into the ticket (or link to this doc) and mark it done once the field, automation, and filters are live — or comment a status update and push the date if it's not fully built by the due date.
