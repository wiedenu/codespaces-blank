GOIZ currently runs GA4 via a direct gtag.js tag — Google Tag Manager is not deployed. That changes the shape of this whole section: without GTM, every custom event below requires a developer hand-coding a gtag('event', ...) call and redeploying, one at a time, indefinitely. The real first ask isn't "which events do we want" — it's getting GTM in place so MOps can configure most future events without a dev ticket every time. Good news on scope: pulled the page source and confirmed the CSP already explicitly allowlists googletagmanager.com and tagmanager.google.com — deploying GTM is a script-tag insertion, not a CSP negotiation with IT.
